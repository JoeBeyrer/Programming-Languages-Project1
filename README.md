# Repository Setup
The Delphi grammar used to generate the lexer and parser can be found in `delphi.g4`.

The `tests` directory contains 10 test files for the base class functionality and 2 bonus test files for the inheritence and interface components of the project.

The java_interpreter includes the generated lexer and parser, all supporting .class files, the Java interpreter in Interpreter.java, and code to support test execution in Main.java.


# Running the Code in a Git Bash Terminal
To run the experiments within this code, the following commands must be executed in a Git Bash terminal:
1. Compile using `javac -cp "java_interpreter/antlr-4.13.2-complete.jar" -d java_interpreter java_interpreter/*.java`
2. Run using `java -cp "java_interpreter;java_interpreter/antlr-4.13.2-complete.jar" Main tests/test1.pas`
  - There are 9 test files located in tests. Please ensure you run this command for test1.pas up to test7.pas. Then run bonustest1.pas and bonustest2.pas for to test for the 20% bonus features.

If there are issues with the .class files located within the java_interpreter directory, run the following command before executing the test commands above:
- `rm java_interpreter/*.class`


If issues continue to persist, the lexer and parser can be regenerated by taking the following actions:
1. Run `java -jar java_interpreter/antlr-4.13.2-complete.jar -Dlanguage=Java -visitor delphi.g4` from a Git Bash terminal
2. Run `mv *.java java_interpreter/`
3. Run `rm java_interpreter/*.class`
4. Run `javac -cp "java_interpreter/antlr-4.13.2-complete.jar" -d java_interpreter java_interpreter/*.java`
5. Run test cases using the `java -cp "java_interpreter;java_interpreter/antlr-4.13.2-complete.jar" Main tests/[TEST_FILE_NAME].pas` command as shown previously.


# Test Descriptions
NOTE: Only integers can be used during testing per the assignment description
- `test1.pas`: constructor with parameter + readln input + private field access
  - Requires integer input in command line to run
  - Outputs the class field created from the user input
- `test2.pas`: no-arg constructor + destructor
  - Outputs 1 after succesfull construction and 0 after destruction
- `test3.pas`: multiple instances of the same class
  - Outputs point coordinates for two instances of the point object (3, 7) and (10, 20)
- `test4.pas`: arithmetic inside constructor body
  - Outputs 20 from 12 + 8 (hard-coded parameters) calculation inside constructor body
- `test5.pas`: multiple different classes in one program
  - Outputs 3 and 9 from fields of  Dog and Cat class instances
- `test6.pas`: readln input + arithmetic in constructor + multiple fields
  - Requires two integer inputs in command line to run
  - Outputs width (first input), height (second input), and area (product of two inputs)
- `test7.pas`: constructor with parameter + destructor explicitly called
  - Outputs 99 (parameter hardcoded in constructor) and 0 upon instance destruction
- `bonustest1.pas`: inheritance + subclass accessing parent field + two-parameter constructor
  - Outputs 4 and 3 (parameters of inherited class)
- `bonustest2.pas`: interface declaration + class implementing interface + procedure method call on instance
  - Outputs 50 (the hardcoded parameter passed to the constructor) using the Print procedure
